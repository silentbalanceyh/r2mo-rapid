# MXT Change Log Rules

> Single-purpose convention for recording meaningful MXT document changes.

## 1. When to Add a Change Log Entry

Add or refresh change-log information when the update is structurally meaningful:

- new route file,
- new boundary or guide file,
- major reading-order change,
- major architecture interpretation change,
- major governance-rule change.

## 2. Suggested Format

```markdown
## Change Log

| Date | File | Change Type | Summary | Trigger |
|------|------|-------------|---------|---------|
| 2026-03-27 | sample.md | add | Created a new guide | New routing need |
```

## 3. Suggested Change Types

- add
- update
- delete
- refactor

## 4. Suggested Triggers

- framework upgrade
- new module family
- abstraction rule change
- documentation review feedback
- retrieval optimization

## 5. Final Rule

Use change-log entries to explain structure changes, not to narrate every wording tweak.
